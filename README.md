# MedConsult — скелет PWA для протокола консультации

React + Vite. Сейчас всё работает на localStorage, Supabase подключается отдельно.

## Что уже есть

- Конструктор жалоб/анамнеза чипами (иерархия с модификаторами) + ручной ввод через запятую
- Автоподсказки по истории ввода жалоб (частота использования)
- Назначения: автоподтягивание препаратов по жалобам (с весами), переключатель
  доказательности (гайдлайн / проверено мной / off-label), кнопка замены на аналог
  внутри фарм. группы
- Мгновенная локальная проверка аллергии при добавлении препарата
  (прямое совпадение, совпадение по группе, известная перекрёстная реактивность) —
  без AI, база в src/data/drugSafety.js
- Переключатель финального текста: по полям / единым текстом, копирование
- Полный экспорт/импорт данных в JSON
- Два стартовых шаблона (первичный приём, повторный приём) — редактируются в
  src/lib/store.js -> seedTemplates()

## Чего пока нет (следующие шаги)

- Supabase: таблицы + RLS ("мои шаблоны/пациенты" vs "публичные шаблоны")
- OpenRouter-вызовы: проверка взаимодействий, причёсывание текста, подсказка диагноза
- Преоперационный эпикриз и протокол операции как отдельные типы шаблонов
- Версионирование шаблонов (snapshot на визит, чтобы правка шаблона не ломала старые протоколы)

## Запуск локально

npm install
npm run dev

## Деплой на GitHub Pages

1. Создай репозиторий на GitHub, например medconsult, запушь туда этот код.
2. В vite.config.js проверь строку base: '/medconsult/' — она должна
   точно совпадать с названием репозитория (регистр важен).
3. В package.json замени homepage на реальный https://<твой-username>.github.io/medconsult
4. Задеплой:
   npm run deploy
   Это соберёт проект и запушит dist/ в ветку gh-pages.
5. В настройках репозитория на GitHub -> Settings -> Pages -> Source: выбери
   ветку gh-pages (появится после первого деплоя).
6. Сайт будет доступен на https://<твой-username>.github.io/medconsult/

## Подключение Supabase (когда понадобится)

Создай .env в корне:
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=xxxx

Пока .env нет — приложение полностью работает офлайн на localStorage,
ничего не ломается.
